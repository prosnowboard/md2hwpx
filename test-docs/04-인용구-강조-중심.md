# Oh-My-ClaudeCode 철학과 원칙

> "고급 사용자는 수년간 zsh를 커스터마이징하지만, 대부분은 그냥 oh-my-zsh를 쓴다. Claude Code를 배우지 마라. 그냥 OMC를 써라."

---

## 핵심 철학

**Oh-My-ClaudeCode**는 단순한 플러그인이 아닙니다. 이것은 *멀티 에이전트 오케스트레이션 시스템*으로, 개발자가 코딩에 집중할 수 있도록 **모든 반복 작업을 자동화**합니다.

### 위임-우선 원칙

> **당신의 역할은 오케스트레이터입니다. 직접 작업하지 마세요.**

이 원칙은 OMC의 모든 행동을 관통합니다:

- 코드 변경은 **절대** 직접 하지 않음 → `executor`에게 위임
- 복잡한 디버깅은 **절대** 직접 하지 않음 → `architect`에게 위임
- UI/프론트엔드 작업은 **절대** 직접 하지 않음 → `designer`에게 위임

> **참고**: 파일 읽기, 상태 확인, 할일 관리 등 간단한 작업은 직접 수행합니다.

### 검증-완료 프로토콜

> ⚠️ **절대 규칙**: 신선한 검증 증거 없이는 완료를 주장하지 마세요.

완료를 주장하기 전에 반드시:

1. **식별**: 이 주장을 증명하는 명령은 무엇인가?
2. **실행**: 검증 명령 실행
3. **확인**: 출력 확인 — 통과했는가?
4. **주장**: 증거와 함께 완료 주장

***경고 신호***:
- ~~"아마 될 거예요"~~처럼 모호한 표현 사용
- 검증 전에 만족감 표현
- 테스트/빌드 실행 없이 완료 주장

### 매직 키워드 조합

**자연어 명령**을 인식해 자동으로 적절한 모드를 활성화합니다. 키워드를 *조합*하면 더 강력합니다:

- `"ralph ulw: migrate database"` = **지속성** + **병렬성**
- `"autopilot: build me a todo app"` = **완전 자율 실행**
- `"eco fix all errors"` = **토큰 절약** + 병렬 처리

> 💡 **팁**: 명령어를 외울 필요 없습니다. 그냥 원하는 것을 말하세요.
> Autopilot이 자동으로 활성화되어 나머지를 처리합니다.
