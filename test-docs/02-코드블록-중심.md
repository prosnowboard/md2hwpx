# Oh-My-ClaudeCode 설치 및 설정 가이드

## 설치 방법

### Oh-My-ClaudeCode 설치

```bash
# 1. 마켓플레이스 추가
/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode

# 2. 플러그인 설치
/plugin install oh-my-claudecode

# 3. 초기 설정
/oh-my-claudecode:omc-setup
```

### Oh-My-OpenCode 설치

```bash
# bunx로 실행
bunx oh-my-opencode

# 또는 npm 설치
npm install -g oh-my-opencode

# 환경 진단
oh-my-opencode doctor
```

## 설정 파일 구조

설정 파일은 다음 위치에 저장됩니다:

```json
{
  "defaultExecutionMode": "ultrawork",
  "hudEnabled": true,
  "hudLayout": "compact",
  "modelPreferences": {
    "simple": "haiku",
    "standard": "sonnet",
    "complex": "opus"
  }
}
```

## Notepad Wisdom 디렉토리 구조

```
.omc/notepads/{plan-name}/
├── learnings.md    # 기술적 발견과 패턴
├── decisions.md    # 아키텍처 및 설계 결정
├── issues.md       # 알려진 이슈와 해결책
└── problems.md     # 차단 요소와 도전
```

## Python 예제: 에이전트 위임

```python
from task import Task

# 간단한 탐색 위임
result = Task(
    subagent_type="oh-my-claudecode:explore",
    model="haiku",
    prompt="Find all API endpoints in the codebase"
)

# 복잡한 분석 위임
analysis = Task(
    subagent_type="oh-my-claudecode:architect",
    model="opus",
    prompt="Analyze the race condition in auth module"
)
```

## TypeScript 예제: 커스텀 훅

```typescript
interface HookConfig {
  name: string;
  trigger: 'pre-commit' | 'post-commit' | 'on-error';
  action: () => Promise<void>;
}

const myHook: HookConfig = {
  name: 'lint-check',
  trigger: 'pre-commit',
  action: async () => {
    console.log('Running lint check...');
    // 실행 로직
  }
};
```

## 인라인 코드 사용 예시

설정 파일은 `~/.claude/.omc-config.json`에 위치합니다. `defaultExecutionMode`를 `"ultrawork"` 또는 `"ecomode"`로 변경할 수 있습니다.

명령줄에서 `uvicorn md2hwpx.server:app --reload`를 실행하면 개발 서버가 시작됩니다.
